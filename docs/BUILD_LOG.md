# Build log and Codex collaboration record

This log distinguishes hackathon-period work and records important human decisions. The project began from an empty repository on **July 18, 2026** during the OpenAI Build Week submission period.

## July 18 — foundation and product correction

Codex contributions:

- Extracted the submission constraints and judging criteria
- Researched the current official GPT-5.6, Responses API, and Structured Outputs guidance
- Developed and compared multiple bounded Education concepts
- Initially scaffolded Counterproof, a programming-misconception prototype
- Researched AI-literacy gaps, beginner build workflows, learning science, GitHub, safety, accessibility, and deployment guidance
- Reframed the product after the human clarified that the goal was deep education about building with AI—not coding education and not a real project generator
- Designed the TRACE curriculum, flagship Repair Café mission, deterministic evidence model, signature interactions, GPT-5.6 debrief boundary, test strategy, and final interface

Human decisions:

- Chose the Education track
- Approved building from an empty repository with Codex as the primary collaborator
- Rejected a learning-to-code direction
- Rejected a real AI website/project generator
- Defined the target learner as someone with no prior AI-building, GitHub, testing, or deployment experience
- Made content quality and the interactive learning experience the top priorities
- Chose one comprehensive mission over multiple shallow lessons
- Required every content item to be useful, applied, and connected to a visible consequence

Key product decisions:

- The authored curriculum, defects, consequences, evidence, and score are authoritative
- GPT-5.6 personalizes the final debrief but cannot set scores, progression, correctness, or ship readiness
- The learner acts before receiving instruction; failed decisions remain repairable
- TRACE—Target, Record, Assign, Check, Evolve—is the transferable learning spine
- Context X-ray, Blast Radius, Proof Ledger, and Build Replay make invisible judgment visible
- One mission may show performance evidence but never claims durable mastery
- Judge mode is fully usable without credentials
- The API key and all OpenAI calls remain server-side
- No real GitHub, deployment, purchase, message, permission, or data mutation occurs

Verified checkpoint:

- Production Next.js build passes
- Cloudflare Workers/Vinext production build passes
- 12 current domain/API tests pass after the product pivot
- 4 Playwright journeys pass across desktop Chromium and a Pixel 7 viewport
- The same complete journeys pass against the local Workers runtime, including the debrief route
- Complete best-practice and meaningful-failure paths are covered
- Dependency audit reports zero known vulnerabilities
